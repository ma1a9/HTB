# Enumeration
1. ユーザー（whoami）、グループ（net user [username]）を確認します。
2. winPEASをfast、searchfast、cmdの各オプションで実行します。
3. Seatbeltや他のスクリプトも実行できます。
4. スクリプトが失敗し、その理由がわからない場合、このコースにある手動コマンドや、オンラインの他の Windows PrivEsc cheatsheets をいつでも実行できます (例: https://github.com/swisskyrepo/PayloadsAllTheThings/blob/master/Methodology%20and%20Resources/Windows%20-%20Privilege%20Escalation.md)
# 戦略
WinPEASや他のツールで興味深い結果が見つかった場合は、それをメモしておいてください。   
 特権昇格の方法に必要なもののチェックリストを作成し、ウサギの穴を避けることができます。   
ユーザーのデスクトップやその他の一般的な場所にあるファイルをざっと探してみてください。(e.g. C:\ and C:\Program Files).   
見つけた興味深いファイルには、特権を拡大するのに役立つ情報が含まれている可能性がありますので、目を通してください。   
レジストリやサービスなど、手順が少ないものから試してみてください。 管理プロセスをよく見て、そのバージョンを列挙し、悪用を探します。 攻撃マシンに転送できる可能性のある内部ポートをチェックします。   
まだ管理者シェルがない場合は、フルエニュメレーションダンプを再読込し、奇妙に思えるものをハイライトします。 これは、見慣れないプロセスやファイル名、またはユーザー名かもしれません。 この段階で、カーネルエクスプロイトについて考え始めることもできます。   
パニックにならないでください 特権エスカレーションは厄介です。 練習すれば完璧になります。 覚えておいてほしいのは、試験の場では、方法を見つけるのに時間がかかるかもしれませんが、試験は常に時間内に完了することを目的としているということです。探し続けてください。
